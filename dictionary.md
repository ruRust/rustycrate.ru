---

Дополнения приветствуются. Для добавления перевода нового термина,
создайте [задачу](https://github.com/ruRust/rustycrate.ru/issues/new) в
репозитории сайта. В ней будет происходить обсуждение.

Вопросы - в [чат](https://gitter.im/ruRust/rust_book_ru).

* **abort**

  прерывание выполнения (программы, кода)
  
* **aliasing**

  задание псевдонимов; в отношении указателей/ссылок - совпадение указателя/ссылки
  
* **arm**

  ветка

* **associated**

  ассоциированная (обычно функция, связанная с типом)

* **borrowing**

  заимствование

* **borrow checker**

  анализатор заимствований

* **box**

  упаковка

* **build**

  сборка

* **capacity**

  ёмкость (в отношении векторов)

* **capture**

  метапеременная, захват метапеременных (в макросах)

* **cleanup**

  освобождение ресурсов

* **closure**

  замыкание (конструкция подобная функции, которую можно хранить в переменной)

    * **closure capture environment**
    
      захват окружения замыканием (возможность замыкания захватывать значения переменных в области видимости)
    
    * **closure type inference**
    
      выведение типа замыкания (процесс определения типа замыкания)
    
* **cons list**

  cons список (cons - construct function, структура данных для пары значений, содержащие внутри следующие пары, для образования списка)

* **coercion**

    приведение
 
    * **deref coercion**
    
        разыменованное приведение (автоматический процесс преобразующий ссылку на исходный тип в ссылку на целевой тип)

    * **implicit deref coercion**
    
        неявное разыменованное приведение

* **consume**

  потреблять (гл.)
    
    * **consumer**
    
      потребитель (сущ.)

* **compilation**

  компиляция

* **сompound**

  сложный, комбинированный (тип, группировка простых типов в сложный - кортежи и массивы)

* **concept**

  понятие

* **concurrency**

  конкурентность (как многопоточность и/или параллельность)

    * **shared-state concurrency**
    
      многопоточное разделяемое состояние
    
* **core team**

  основная группа
  
* **coroutine**

  корутина

* **crash**

  аварийно завершиться
  
* **crate**

  крейт (бинарный или библиотечный) 

    * **library crate**
    
       библиотечный крейт
    
    * **binary crate**
    
      двоичный, бинарный, исполняемый крейт (зависит от контекста)
    
* **dangling**

  не действительная (обычно ссылка, указатель - reference, pointer)

* **derived**

  выводимые, наследуемые (типажи traits)

* **dereferencing**

  разыменование (ссылки, указателя)

* **destructure**

  деструктуризация (структур, перечислений, кортежей и ссылок, чтобы использовать разные части этих значений)

    * **destructure (nested) struct**
    
      деструктуризация (вложенной) структуры
    
    * **destructure enum, tuple**
    
      деструктуризация перечисления, кортежа
    
* **dispatch**

  диспетчерезация, связывание

    * **dynamic dispatch**
    
      динамическая диспетчерезация, связывание

* **drop, dropping**

  автоматическое удаление, дроп (в Rust автоматическая очистка ресурсов при выходе переменной из области видимости)

* **encapsulation**

  инкапсуляция (скрытие деталей внутренней реализации)

* **error**

  ошибка

    * **error handling**
    
      обработка ошибок
    
    * **unrecoverable error**
    
      неустранимая, неисправимая ошибка (обычно после panic!)

    * **recoverable error**
    
      устранимая, исправимая ошибка (для Result типа)

    * **propagating errors**
    
      распространение ошибок (передача ошибок из более низких уровней кода в верхние для их дальнейшей обработки)

    * **? operator**
    
      оператор распространения ошибки

* **expand**

  развернуть

* **expansion**

  развёртывание
  
* **expression**

  выражение

* **exhaustive**

  полное, исчерпывающиее (обычно сопоставление)

* **feature**

  возможность
  
* **future**

  футура

* **generic**

  обобщённый

    * **generic type**
    
      обобщённый тип

    * **generic trait**
    
      обобщённый типаж

    * **generic lifetimes**
    
      обобщённое время жизни

* **glob**

  глобальный

    * **glob operator**
    
      оператор глобального импорта (подключение в область видимости всех общих элементов, определённых по указаному пути)

* **goroutine**

  горутина  
  
* **inheritance**

  наследование (механизм в ООП, в Rust отсутствует наследование типов)
  
* **iterator**

  итератор (конструкция, способ обработки последовательности из элементов)
  
* **handle**

  дескриптор

* **immutable**

  неизменяемый

* **irrefutable pattern**

  безусловный образец

* **length**

  длина (в отношении векторов)

* **lifetime**

  время жизни

    * **lifetime elision**
    
      неявное выведение времени жизни (с помощью анализатора заимствований в стандартных ситуациях без необходимости явного указания аннотаций)
    
    * **lifetime elision rules**
    
      правила неявного выведения времен жизни
    
    * **static lifetime**
    
      статическое время жизни
    
* **lint**

  статический анализ / статическая проверка (в зависимости от контекста)

* **literal**

  литерал, литеральный
 
* **loop**

  цикл
 
    * **conditional loop**
    
      условный цикл
     
* **macro, macros**

  макрос (макроопределение)

    * **declarative macros**
    
      декларативный макрос (макроопределение)

    * **procedural macros**
    
      процедурный макрос (макроопределение)

    * **attribute-like macros**
    
      подобный атрибутам макрос

    * **function-like macros**
    
      функционально подобный макрос

* **mangling**

  манглинг, искажение (процесс подмены имени функции другим именем для внутреннего удобства компилятором)

* **match**

  сопоставление с образцом (гл.), сопоставление (сущ.)

    * **match guard**
    
      дополнительное условие (доп. условие if в конструкции ветки match) 
    
    * **match binding**
    
      оператор связывания (позволяет создать переменную со значением, которое мы одновременно сравниваем со значением соответствия шаблону) 
    
* **matcher**

* **matching**

    сопоставление (обычно с образцом, шаблоном)

    * **matching literal**
    
        сопоставление литерала
    
    * **matching named variable**
    
        сопоставление именованной переменной
    
    * **matching ranges**
    
        сопоставление диапазонов
    
* **metadata**

    метаданные (данные описывающие другие данные)

* **module**

  модуль, раздел внутри крейта

* **monomorphization**

  мономорфизация (процесс превращения обобщённого кода в конкретный код, с заполнением конкретными типами при компиляции)

* **mutable**

  изменяемый (переменная, ссылка)

* **mutability**

  изменяемость, возможность изменить значение (например, у переменной)

    * **interior mutability**
    
      внутренняя изменяемость (шаблон в Rust, позволяющий изменять данные, даже если ссылки на эти данные неизменяемые)

* **multithreading**

  многопоточность

* **native libraries**

  нативные библиотеки (имеются ввиду родные системные библиотеки, например, libc)

* **NLL(Non-Lexical Lifetime)**

  нелексическое время жизни

* **operator**

  оператор

    * **dereference operator**
    
      оператор разыменования (ссылки)
        
    * **operator overloading**
    
      перегрузка оператора (реализация пользовательское поведения некоторого оператора из стандартной библиотеки)
        
* **ownership**

  владение, владение ресурсом, право владения

    * **ownership transference**
    
      передача владения
    
    * **multiple ownership**
    
      множественное владение
    
* **package**

  пакет (объединение крейтов в Rust, может содержать только одни библиотечный крейт и ноль или много исполняемых)

* **panic**

  паника (программы, кода, вызванная выполнением макроса panic! и прерыванием выполнения кода)

* **path**

  путь (как способ наименования, обращения, ссылки к элементам бибилиотеки крейта)

* **pattern** (отдельно стоящее, как "pattern" в левой части "let")

  образец

* **pattern matching**

  сопоставление с образцом

* **parallelism**

  параллелизм
  
* **parse**

  проводить разбор
  
* **polymorphism**

  полиморфизм
  
* **pointer**

  указатель 
  
    * **smart pointer**
    
      умный указатель (специальные типы, конструкции для переменных, содержащие указатель на память со специальными возможностями) 
      
    * **raw pointer**
    
      сырой указатель (для исподьзования в небезопасном коде) 
      
* **profile**

  профиль (предопределенный набор настроек)
  
    * **release profile**
    
      профиль релиза
      
    * **build profile**
    
      профиль сборки
      
    * **development profile**
    
      профиль разработки
      
* **publishing**
  
  публикация (размещение крейта на сетевых ресурсах)
  
* **range**
  
  диапазон
  
  * **inclusive range**
  
    замкнутый диапазон
  
  * **exclusive range**
  
    полуоткрытый диапазон
  
* **reference**

    ссылка, cсылочная переменная

    * **reference сycle**
    
        ссылочное зацикливание (ситуация в которой возможна утечка памяти)
    
    * **reference counting**
    
        подсчет ссылок
    
* **refutability**

  возможность опровержения (опровержимые *refutable* и неопровержимые *irrefutable* образцы, шаблоны) 
  
* **safety**

  безопасность (возможности Rust обеспечивающие безопасное использование памяти, безопасную многопоточность, отсутствие гонок данных и др.) 
  
    * **type safety**
    
      безопасность типа (возможности Rust обеспечивающие безопасное использование памяти, безопасную многопоточность, отсутствие гонок данных и др.) 
      
* **scalar**

  скалярный (тип), единичное значение, обычно для целых, вещественных чисел, логический тип и символ 
  
* **separation of concerns**

  разделение ответственности 
  
* **shadowing**

  затенение

* **slice**

  срез

* **statement**

  оператор
  
* **shutdown**

  завершение (обычно успешное)
  
* **test**

  тест

    * **test organization**
    
      организация тестов

    * **unit test**
    
      модульный тест

    * **integration test**
    
      интеграционный тест

    * **test-driven development**
    
      разработка на основе тестов

* **thread**

  поток

    * **single-threaded**
    
      однопоточный
    
    * **multithreaded**
    
      многопоточный
    
    * **thread pool**
    
      пул потоков
    
* **throughput**

  пропускная способность

* **token**

  токен

* **tokenize**

  разбить на токены

* **token tree**

  дерево токенов

* **trait**

  типаж

    * **trait object**
    
      типаж-объект
    
    * **trait bound**
    
      ограничение, граница типажа (специальный синтаксис перечисления типажей)
    
    * **multiple trait bounds**
    
      несколько ограничений, границ типажа (специальный синтаксис для перечисления нескольких ограничений)
    
    * **definition a trait**
    
      объявление типажа
    
    * **implementing a trait**
    
      реализация типажа (у типов)
    
    * **default trait implementation**
    
      реализация типажа по умолчанию (реализация методов типажа внутри типажа, для переиспользования готового кода в типе, реализующим данный типаж)
    
* **tuple struct**

  кортежная структура

* **variable**

  переменная

    * **variable binding**
    
      имя (связанное имя) / значение (связанное значение), привязка (в зависимости от контекста)
    
    * **environment variable**
    
      переменная окружения (обычно внешняя по отношению к программе переменная, переменная из ОС)
    
* **type**

  тип

    * **unit type**
    
      единичный тип

    * **never type**
    
      пустой тип

    * **dynamically sized type**
    
      динамический тип

    * **recursive type**
    
      рекурсивный тип (тип описывающий рекурсивные конструкции)
    
    * **non-recursive type**
    
      не рекурсивный тип
    
    * **type synonym**
    
      синоним типа ("переименование" типа)
    
    * **type aliase**
    
      псевдоним типа (делается с помощью синонима типа)
    
* **unsafe**

  небезопасный (небезопасный код в Rust, предоставляющий дополнительные возможности)
    
    * **unsafe function, method, trait**
    
      небезопасная функция, метод типаж (небезопасные, дополнительные возможности)

* **unwinding**

  раскрутка стека (в результате ошибки)

* **yank, yanking**

  выламывание (пометка версии крейта, который нельзя использовать в качестве новой зависимости в проекте)

* **workspace**

  рабочее пространство проекта (возможность управления несколькими связанными пакетами, разработанными в рамках одного проекте)
